# Baseline Calibration — алгоритм перекалибровки

Этот файл — extract из Step 2.6 `end-session/SKILL.md`. Применяется, когда после очередного `STATS_ROW` нужно пересчитать `Baseline` колонку в `~/.claude/stats/doc-writer.md` → `## Compound Effect`.

## Формула compound effect

```
effect_% = round(((baseline − avg_iter) / baseline) * 100)
```

- `avg_iter` — среднее значение поля `iterations` (= `Шагов: N` из COMPLETE-промпта) по всем `STATS_ROW` в `## Session Log` для данной complexity.
- `baseline` — «сколько заняла бы задача без системы», начально хардкод: **Trivial=1, Standard=2, Complex=4**.

## Источник `iterations` в `STATS_ROW`

- Поле **берётся из** `Шагов: N`, которое главный агент передаёт в промпте `@doc-writer COMPLETE`.
- Правила подсчёта `N` — глобальный `~/.claude/CLAUDE.md` → секция «Подсчёт реальных шагов задачи».
- End-session **не пересчитывает** iterations сам — только агрегирует значения из `## Session Log` в `avg_iter`.
- Если в STATS_ROW суффикс `(approximate)` — counter использовал fallback `grep -c "### Итерация "` (= частота делегирования, не реальные шаги). Помечать такие записи и **не давать им вес в калибровке**.

## Триггер перекалибровки

Срабатывает при **каждом** обновлении stats, проверяется автоматически:

- ≥ 50 задач **всего** в `## Session Log` (по записям без `(approximate)`), ИЛИ
- ≥ 20 задач в одном уровне complexity (по записям без `(approximate)`).

## Алгоритм перекалибровки

1. Перечитать все `STATS_ROW` из `## Session Log` для данной complexity.
2. **Исключить записи с `(approximate)`** — там counter измерял делегирование, а не реальные шаги (искажает p75).
3. Если после фильтра < 20 точных записей — отложить калибровку, пометить:
   ```
   ⚠️ baseline calibration deferred: <N> точных записей < 20
   ```
4. Новый baseline = **p75 (75-й перцентиль) iterations** (округление вверх до целого).
5. Обновить колонку `Baseline` в таблице `## Compound Effect`.
6. Обновить `_Last updated:_` в шапке файла.
7. Дописать комментарий в шапку:
   ```
   <!-- Recalibrated: baseline <Trivial|Standard|Complex> N→M on YYYY-MM-DD (basis: <K> точных записей) -->
   ```

## Почему p75, а не медиана / mean

Baseline — это «сколько заняла бы задача **без** системы», а не средняя по типичной задаче. p75 ловит верхнюю часть распределения — задачи, где compound effect должен максимально проявляться. Медиана занижает, mean чувствителен к выбросам.

## Когда сбрасывать стек полностью (reset, не калибровка)

**Triggers** для полного сброса `## Session Log` (а не пересчёта):

- Проект меняется принципиально — новая платформа, новый язык, миграция на новую архитектуру.
- Старые данные не применимы к новой природе задач.

**Действие:** очистить `## Session Log` и начать копить заново с хардкод-baseline (Trivial=1, Standard=2, Complex=4). В шапку дописать:

```
<!-- Reset on YYYY-MM-DD: project shifted to <new context>, old log archived to doc-writer-archive-YYYY-MM-DD.md -->
```

## Why калибровка важна

Baseline отражает **реальное распределение** задач пользователя, не гипотетические константы. Без калибровки через полгода compound effect становится недостоверным: задачи усложняются, agents обучаются, а baseline остаётся прежним — числа искажаются.
