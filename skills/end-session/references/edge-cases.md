# End Session — Edge Cases

Этот файл — extract из секции `## Edge cases` в `end-session/SKILL.md`. Описание нестандартных ситуаций, в которые попадает скилл, и того, как реагировать.

## 1. Не git-репозиторий

Шаги, требующие git: 2.1, 2.2 (частично), 5.1.

**Действие:** пропустить с пометкой `✅ N/A (no git)`. Остальное прогнать как обычно.

Типичный кейс — пользователь работает в каталоге с конспектами, исследовательскими файлами, без `git init`. Скилл всё ещё может проверить субагентов, doc-writer, рекомендации.

## 2. Сессия без задачи (просто болтовня / исследование)

Скилл всё равно стоит запустить — он сам определит, что:
- `✅ no changes` (git clean),
- `✅ no agents` (специалистов не было),
- `✅ no doc needed` (правил DoD не активировано),
- Step 4 = `✅ no recommendations`.

Завершение через ~30 секунд. Это нормальный исход, не пугаться.

**Why оставлять запуск:** end-session — это часть ритуала «закрытия контекста», и его прогон даже на пустой сессии формирует привычку и тренирует main-agent на корректную классификацию.

## 3. Скилл вызвали при недописанной задаче

Признаки:
- Есть незакоммиченные правки.
- В коде явные `TODO:` / `FIXME:` без anchor (см. Step 2.9).
- Главный агент сам понимает, что задача не завершена (есть `// implement me`, undefined fields, незакрытые блоки).

**Действие:**
- Вердикт `❌ NOT READY`, перечислить что осталось.
- **Не** пытаться «дозавершить» — это работа главного агента, а не скилла.
- Возвращать управление главному агенту, чтобы тот закончил задачу.

## 4. `@doc-writer` уронил / не ответил

Признаки:
- Subagent task завершился с ошибкой.
- Timeout (нет ответа в течение длительного времени).
- Ответ есть, но не содержит ожидаемых маркеров (`STATS_ROW`, `INDEX_ROW`, или фразы про завершение документа).

**Действие:**
1. Отметить `⚠️ doc-writer failed` с прямой ссылкой на текст ошибки.
2. Предложить пользователю через `AskUserQuestion` 3 опции:
   - **Запустить ещё раз** — retry в новом subagent task.
   - **Пропустить и закрыть руками** — пометка `⚠️ doc-writer skipped (manual close)`, в финальный отчёт добавить TODO для пользователя.
   - **Отложить** — `⚠️ doc-writer deferred`, остановить end-session, вернуть управление.
3. **Не зацикливаться на ретраях** — максимум 1 повтор после ошибки; если и он упал — обязательно спросить пользователя.

## 5. Graphify CLI установлен, но `graphify-out/` нет

Проект ещё не проиндексирован.

**Действие:** предложить пользователю первичную индексацию через `AskUserQuestion` (3 опции):

| Опция | Действие |
|---|---|
| `Run /graphify . now` | Пользователь сам запустит (требует LLM-токенов) |
| `Skip for this session` | Пометка `⚠️ no graph (deferred)`, продолжаем |
| `Never ask for this project` | Запись в project memory `graphify_optout.md`, больше не предлагать |

**Сам не запускать** `/graphify .` — это требует LLM-токенов и решения пользователя по бюджету.

Если опция «никогда» уже зафиксирована в memory — пропустить шаг тихо, без `AskUserQuestion`.

Подробности — `references/graphify-freshness.md`.

## 6. Несколько задач за сессию

Сессия содержала 2+ логических задач (например, сначала bug-fix, потом feature, потом рефакторинг).

**Действие:** прогон должен покрыть **последнюю** «логическую задачу» (последний Prompt Contract).

Если пользователь хочет проверить всю сессию целиком — запросить уточнение через `AskUserQuestion`:

| Опция | Действие |
|---|---|
| **Проверить только последнюю задачу** | По умолчанию. Стандартный прогон gate. |
| **Проверить все задачи последовательно** | Прогнать gate N раз, отчёт по каждой отдельно, общий verdict — худший из всех |
| **Только финальный коммит и общий статус** | Skip subagent sanity / doc-writer / recommendations, проверить только `git status` + наличие коммита |

## 7. Subagent task всё ещё работает в фоне

Признаки:
- `@doc-writer` запущен с `run_in_background: true` и ещё не вернул результат.
- Главный агент дошёл до Step 5 быстрее, чем фоновый агент закончил.

**Действие:**
1. **Дождаться** уведомления о завершении фонового агента.
2. После уведомления — обработать `STATS_ROW`/`INDEX_ROW` (Steps 2.6/2.7).
3. Если фоновый агент завис надолго — таймаут 2 минуты, после — пометка `⚠️ doc-writer hung`, спросить пользователя.

**Не блокировать диалог** долго: если ответ затягивается, дать пользователю промежуточный статус и спросить, ждать ли.

## 8. `STATS_ROW` есть, но его формат сломан

Ожидаемый формат:
```
STATS_ROW: | YYYY-MM-DD | project | complexity | iterations | effect_% | doc-link |
```

Если формат отличается (например, отсутствует pipe-разделитель, не хватает колонок, лишние поля):

**Действие:**
- НЕ пытаться парсить — может попортить таблицу.
- Пометить `⚠️ STATS_ROW malformed: <verbatim>`, в финальный отчёт включить точный текст.
- Спросить пользователя через `AskUserQuestion`:
  - **Skip this entry** — не записывать, продолжить.
  - **Manually fix and add** — открыть `~/.claude/stats/doc-writer.md`, дать пользователю править.
  - **Append as-is (might break formatting)** — записать в `## Session Log` с пометкой `<!-- malformed -->`.

## 9. Рекомендация из Step 4 указывает на out-of-scope plugin

Если recommendation предлагает менять Anthropic-managed plugins/skills (`compound-engineering:*`, `vercel:*`, `cloudflare:*`, `figma:*`, `amplitude:*`):

**Действие:**
- НЕ записывать в queue (даже как LOW).
- В summary показать строку `⊘ X recs rejected (out-of-scope)`.
- Это страховка от мусора в queue.

Подробности — `~/.claude/recommendations/README.md` → секция «Best-practice критерии классификации».

## 10. Auto-commit упал на pre-commit hook

`/commit` вернул non-zero exit, в stderr — текст pre-commit hook'а (detekt, lint, type-check).

**Действие:**
- Пометить `❌ auto-commit failed: pre-commit hook`.
- **Не** заявлять задачу завершённой, не игнорировать `--no-verify`.
- В финальный отчёт включить:
  - Точный текст ошибки.
  - Файлы, на которые ругается hook.
  - Конкретные исправления (если очевидны).
- Передать управление пользователю, чтобы тот исправил и повторил.

**Запрещено:** ставить `--no-verify` или другие байпасы хука без явного запроса пользователя (см. глобальный `CLAUDE.md` → git-protocol).
